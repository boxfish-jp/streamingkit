# 接続状況確認

## 何がしたい？
- ダウンディテクター的なサービスが欲しい。  
- 配信クライアントが立ち上がっていない状態のときに、配信を始めた場合に通知するといった機能が欲しい。  

### ダウンディテクターで何がしたい？
- 視聴者目線でふぐおの配信が正常でないときに、ふぐおのせいなのか、おまかんなのか知りたい。  
- 配信ツール機能が正常に動かないときに、何が原因なのか知りたい。
(ptptしてるときにニコ生側なのか自分側なのかお知らせしてくれるの？)

## どういうふうに知らせるか
- Webサイトを用意する。  
- Cloudflareのアレなイメージ
- 気になるトピックごとにそれを作ったほうがいいかも
  - 音声: ローカル音声→OBS→ニコ生サーバー音声
  - 映像: OBS→ニコ生サーバー音声
  - コメント読み上げ: コメント取得→音声合成エンジン→音声再生
  - コマンド動画再生: コメント取得→エフェクト

## 必要なデータ
### コメント読み上げ
- 配信が開始しているか
- コメント取得できているか
- 直近の音声合成でエラーが起きているか
- オーケストレーターが起動しているか
- クライアントが起動しているか
- クライアントとオーケストレーター間が接続ができているか
- 音声再生できるか

### 動画再生
- コメント取得できているか
- オーケストレーターが起動しているか
- エフェクトが起動しているか
- クライアントとエフェクト間が接続ができているか

### 音声
- ふぐおが喋っているか
- OBSマイク入力の情報
- ニコ生の配信音声

### 映像
- OBSの映像が配信されているか
- ニコ生の配信映像が配信されているか

## 設計
### コメント読み上げ
- 配信が開始しているか: 定期的にメッセージを出す
- コメント取得ができているか: 
- 直近の音声合成でエラーが起きているか(エラーを見張る)
- 接続できているかについては、ハブからpingを投げて、接続しているクライアントがpongを返す.
- オーケストレーターとクライアントが起動しているか: Web上に投げる。  
- 音声再生できるか
